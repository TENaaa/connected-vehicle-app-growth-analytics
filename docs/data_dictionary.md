# 数据字典

所有表均为合成数据，只使用虚拟 ID。

## 核心实体

| 表 | 粒度 | 用途 |
| --- | --- | --- |
| `users` | 每个合成车主一行 | 注册日期、城市层级、获客渠道、生命周期阶段、营销许可。 |
| `vehicles` | 每个合成车主车辆一行 | 车型分层、能源类型、购车时间、车龄分桶、高阶辅助驾驶标记、剩余质保月数。 |
| `user_features` | 每个用户一行 | App 使用、出行、远控、驾驶/能耗评分、续约/服务到期、产品亲和度和流失风险。 |

## 产品与活动

| 表 | 粒度 | 用途 |
| --- | --- | --- |
| `products` | 每个产品一行 | 产品类别、合成价格、成本、订阅月数、是否组合包。 |
| `campaigns` | 每个增长活动一行 | 目标生命周期阶段、主推产品、主触达渠道、预算、定向规则。 |
| `content_items` | 每个内容资产一行 | 抽象的 App 卡片、Push、短信、坐席话术或经销商消息，并关联产品/活动。 |

## 行为与结果

| 表 | 粒度 | 用途 |
| --- | --- | --- |
| `experiment_assignments` | 每个用户一行 | holdout/control/treatment 实验分组和分组日期。 |
| `touchpoints` | 每次触达或触达尝试一行 | 渠道、活动、产品、时间、送达状态、成本、触达压力标记。 |
| `events` | 每个行为事件一行 | 登录、功能使用、曝光、点击、购买意向、创建订单、退订。 |
| `orders` | 每个合成订单一行 | 产品、活动、归因触点、订单状态、收入、成本、毛利。 |
| `subscriptions` | 每个订阅周期一行 | 产品开始/到期日期、续约窗口、状态、自动续约标记。 |

## 运营与治理

| 表 | 粒度 | 用途 |
| --- | --- | --- |
| `call_center_contacts` | 每次坐席触达一行 | 触达状态、通话时长、结果、坐席等级。 |
| `dealer_leads` | 每条经销商/服务线索一行 | 线索类型、SLA 小时、状态、转化订单引用。 |
| `data_quality_checks` | 每个合成检查一行 | 检查状态、检查行数、失败行数、失败率。 |

## ID 与隐私策略

ID 均为虚拟字符串，例如 `U000001`、`V000001`、`O00000001`。数据集刻意排除姓名、联系方式、车辆唯一标识、车牌、精确地址、真实门店、真实人员、真实组织和真实内部系统标识。

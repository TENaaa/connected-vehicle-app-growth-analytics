# 指标体系

## 北极星指标

**30 日服务活跃率**：注册车主中，在注册后 30 天内使用核心功能、产生购买意向或创建订单的用户占比。

这个指标比单纯登录率更贴近车主 App 的业务价值，因为车联网服务的目标不是让用户打开 App，而是让用户完成远程功能使用、服务意向、续约、保养、保障、补能或权益相关行为。

## 生命周期漏斗

| 层级 | 指标 | 定义 |
| --- | --- | --- |
| 注册 | 注册用户数 | 进入车主 App 旅程的合成用户。 |
| 激活 | D7 登录 | 注册后 7 天内至少登录一次。 |
| 核心价值 | D30 核心功能 | 注册后 30 天内至少产生一次核心功能事件。 |
| 触达 | 活动触达 | 至少有一次成功送达的活动触点。 |
| 意向 | 购买意向 | 至少有一次购买意向事件。 |
| 转化 | 创建订单 / 付费订单 | 合成数据中的商业化结果。 |
| 续约 | 续约窗口活跃 | 处于续约相关窗口且订阅状态仍活跃。 |

## 增长与商业化

| 指标 | 用途 |
| --- | --- |
| 收入 | 衡量不同产品和渠道的商业规模。 |
| 毛利 | 避免只看高收入、忽略低毛利产品。 |
| 渠道 ROI | 比较触达成本与产出价值。 |
| 客单价 | 帮助判断应该做低成本广覆盖，还是高成本精细触达。 |

## 实验设计

合成实验包含三个组：

- `holdout`：自然需求基线，基本不接受活动触达。
- `control`：普通生命周期活动触达。
- `treatment`：个性化多触点策略。

主指标：30 日付费转化率。

Guardrail：Push 退订、过度触达、运营 SLA、数据质量。

## 归因分析

项目比较四种归因视角：

- 首触归因：适合观察需求生成渠道。
- 末触归因：适合观察成交收口渠道。
- 线性归因：把贡献平均分配给路径上的触点。
- 位置归因：给首触和末触更高权重，同时保留中间触点贡献。

归因是描述性分析，不能替代 holdout 增量实验。

## 意向模型

模型预测用户未来 30 日付费转化倾向，并支持人群排序。实际使用时应作为触达优先级工具，并在触达前套用退订、频控和过度触达 guardrail。
